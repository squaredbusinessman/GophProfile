# GophProfile

GophProfile это микросервис для управления аватарками пользователей.

Пользователь загружает свою фотографию в GophProfile один раз. После этого
сторонние платформы, например блоги, форумы, сервисы комментариев и другие
приложения, могут запросить аватарку по email пользователя.

Если пользователь с таким email существует, сервис возвращает его аватарку.
Если пользователь не найден, сервис возвращает стандартное изображение-заглушку.

## Цель проекта

GophProfile решает практическую продуктовую задачу: централизованную загрузку,
хранение, обработку и раздачу пользовательских аватарок через REST API.

Проект строится на востребованном бэкенд-стеке:

- Go
- PostgreSQL
- S3-совместимое объектное хранилище
- RabbitMQ или Kafka
- Docker
- Kubernetes
- Prometheus
- Grafana
- Loki, ELK или OpenSearch
- Jaeger

## Планируемые возможности

- Загрузка аватарки пользователя
- Получение аватарки по email
- Возврат стандартной заглушки для неизвестных пользователей
- Валидация и обработка изображений
- Хранение аватарок в S3-совместимом хранилище
- Хранение пользовательских метаданных в PostgreSQL
- Фоновая обработка задач через брокер сообщений
- REST API для загрузки и получения аватарок
- Контейнеризированное локальное окружение
- Манифесты для развертывания в Kubernetes
- Метрики, логи, трейсы и дашборды

## Начальный API

Точный контракт API будет уточняться по ходу разработки. На старте сервис
планирует поддерживать следующие эндпоинты:

```text
POST /api/v1/avatars
GET  /api/v1/avatars/{email}
GET  /healthz
GET  /readyz
```

## Разработка

Требования:

- Go 1.26.3 или новее

Запуск тестов:

```bash
go test ./...
```

Запуск сервиса:

```bash
go run ./cmd/gophprofile
```

Исполняемая точка входа будет добавлена на следующем этапе реализации.
