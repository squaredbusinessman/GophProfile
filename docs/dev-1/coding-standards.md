# Стандарты кода и документации

## Комментарии к функциям

Все функции проекта должны иметь комментарий на русском языке.

Правила:

- Комментарий ставится непосредственно перед функцией
- Комментарий кратко объясняет назначение функции
- Комментарий не должен заканчиваться точкой
- Для экспортируемых функций комментарий начинается с имени функции, если это
  не ухудшает читаемость
- Комментарии не должны пересказывать очевидную реализацию построчно

Пример:

```go
// Load читает конфигурацию приложения из переменных окружения
func Load() Config {
    // ...
}
```
